## Решение

Переход на SFU (Selective Forwarding Unit) архитектуру **отложен** до момента расширения проекта.

## Текущее состояние

✅ **Mesh-топология работает отлично для текущих потребностей:**
- Поддержка 2-10 участников: ✅ Отлично
- Поддержка 10-20 участников: ✅ Работает
- Одновременное общение с несколькими участниками: ✅ Реализовано

## Когда вернуться к SFU

### Триггеры для пересмотра решения:

1. **Количество участников в комнатах регулярно превышает 20**
   - Пользователи жалуются на проблемы с производительностью
   - Высокая нагрузка на клиентов

2. **Планируется поддержка больших групп (50+ участников)**
   - Новые требования к функциональности
   - Расширение целевой аудитории

3. **Проблемы с масштабированием**
   - Технические проблемы с текущей mesh-топологией
   - Необходимость оптимизации ресурсов
